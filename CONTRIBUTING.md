# Contributing to H.O.P.E.

First off, thanks for taking the time to contribute!

## How to Contribute

1.  **Fork the repository** on GitHub.
2.  **Clone** your fork locally.
3.  **Create a branch** for your feature or bug fix: `git checkout -b my-new-feature`.
4.  **Make your changes**. Ensure you follow the coding standards and add tests if applicable.
5.  **Run tests** locally to make sure everything is working:
    - Backend: `cd src/HOPE.Backend && npm test`
    - Desktop: Run tests via Visual Studio or `dotnet test`
    - AI: `pytest src/HOPE.AI`
6.  **Commit your changes** with a descriptive commit message.
7.  **Push** to your fork: `git push origin my-new-feature`.
8.  **Submit a Pull Request** to the `main` branch.

## Coding Standards

- **C#**: Follow standard .NET conventions.
- **TypeScript**: Use ESLint/Prettier as configured in the project.
- **Python**: Follow PEP 8.

## Reporting Bugs

Please use the Bug Report issue template to report bugs. Include as much detail as possible.
